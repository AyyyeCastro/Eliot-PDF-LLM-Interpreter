Eliot leverages the capabilities of several powerful language models from Hugging Face, including the FLAN-T5 family (small, base, large, xl, xxl), BART (facebook/bart-large-cnn, lidiya/bart-large-xsum-samsum), DistilBART (sshleifer/distilbart-cnn-12-6) for summarization tasks, and DeepSeek-R1-Distill-Qwen-1.5B and the Qwen 1.5 series (Qwen/Qwen2.5-14B) for more open-ended reasoning and analysis.

Eliot is built using a combination of technologies:

  • Python: The core logic and backend of Eliot is written in Python. This includes the PDF processing (using PyMuPDF/fitz), interaction with the Hugging Face Transformers library and prompt engineering, and the overall application flow.
  • Flask: Flask is a lightweight Python web framework used to create the web application interface. It handles requests from the user (like uploading a PDF and selecting a model), processes them using the Python backend, and returns the results to the user's browser.
  • JavaScript: JavaScript is used for front-end interactivity. This includes things like the dynamic sidebar behavior, animations, and handling user input in the browser. It's the bridge from the Python backend to what the user interacts with!
  • HTML/CSS: The structure and styling of the web pages are defined using HTML and CSS. Bootstrap (a CSS framework) is used for responsive design and pre-built components for efficient and modern integration.
